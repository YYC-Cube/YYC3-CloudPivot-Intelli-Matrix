### 失败测试分析

#### 🔴 主要失败原因

**1. React Router Context 缺失 (约 80+ 失败)**
- **错误**: `useNavigate() may be used only in the context of a <Router> component.`
- **影响文件**:
  - `useKeyboardShortcuts.test.tsx`
  - `useWebSocketData.test.tsx`
  - `useOperationCenter.test.tsx`
  - `usePatrol.test.tsx`
  - `useTerminal.test.tsx`
- **原因**: 测试中使用了 `useNavigate` 但没有提供 Router context
- **修复方案**:
  ```typescript
  // ❌ 错误
  render(<Component />);
  
  // ✅ 修复
  render(
    <MemoryRouter>
      <Component />
    </MemoryRouter>
  );
  ```

**2. i18n Context 缺失 (约 30+ 失败)**
- **错误**: `useI18n() may be used only in the context of a <I18nContext.Provider> component.`
- **影响文件**:
  - `useKeyboardShortcuts.test.tsx`
  - `useWebSocketData.test.tsx`
  - 多个组件测试
- **修复方案**:
  ```typescript
  // ✅ 添加 i18n mock provider
  const wrapper = ({ children }) => (
    <I18nContext.Provider value={mockI18nValue}>
      {children}
    </I18nContext.Provider>
  );
  render(<Component />, { wrapper });
  ```

**3. WebSocket Mock 问题 (约 10+ 失败)**
- **错误**: WebSocket 相关测试失败
- **影响文件**:
  - `useWebSocketData.test.tsx`
- **修复方案**:
  ```typescript
  // ✅ Mock WebSocket
  vi.mock('some-websocket-lib', () => ({
    WebSocket: vi.fn(),
  }));
  ```

**4. 异步操作未正确处理 (约 9+ 失败)**
- **错误**: 测试超时或状态未更新
- **影响文件**:

- **修复方案**:
