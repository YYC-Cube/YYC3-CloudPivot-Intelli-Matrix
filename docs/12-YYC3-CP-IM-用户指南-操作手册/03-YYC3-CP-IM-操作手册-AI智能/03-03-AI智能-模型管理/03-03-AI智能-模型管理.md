## 操作步骤

### 1. 上传新模型

**步骤**：
1. 进入"模型管理"页面
2. 点击右上角"添加模型"按钮
3. 填写模型信息：
   - 模型名称
   - 模型版本
   - 模型类型
   - 模型文件（支持 .gguf, .safetensors, .bin 格式）
4. 选择部署 GPU：
   - GPU-A100-01
   - GPU-A100-02
   - GPU-H100-01
5. 点击"部署"按钮
6. 等待模型加载完成

**提示**：
- 70B 参数模型大约占用 140GB 显存
- 推荐使用 A100 80GB 或 H100 80GB

### 2. 查看模型状态

**状态类型**：
- **已部署**：模型已加载到 GPU，可正常使用
- **部署中**：模型正在加载中
- **待命**：模型已部署但暂未使用
- **失败**：模型部署失败

**查看详情**：
1. 点击模型卡片
2. 查看详细信息：
   - 模型大小
   - 分配的 GPU
   - GPU 使用率
   - 显存占用
   - 请求次数
   - 平均响应时间

### 3. 更新模型

**步骤**：
1. 找到要更新的模型
2. 点击"编辑"按钮
3. 上传新版本的模型文件
4. 点击"更新"按钮
5. 选择更新策略：
   - 滚动更新（推荐）
   - 蓝绿更新
   - 立即更新

### 4. 卸载模型

**步骤**：
1. 找到要卸载的模型
2. 点击"删除"按钮
3. 确认删除
4. 模型从 GPU 卸载，释放显存

**警告**：卸载模型会导致正在使用该模型的请求失败

## 模型监控

### 1. 实时监控

在模型管理页面，实时显示：
- GPU 使用率
- 显存占用
- 请求 QPS（每秒查询数）
- 平均响应时间
- 错误率

### 2. 性能分析

点击模型卡片，查看详细性能分析：
- 吞吐量（tokens/秒）
- P50/P95/P99 延迟
- 请求分布图
- 错误类型分布

### 3. 资源优化建议

AI 自动分析模型资源使用，提供优化建议：
- 显存优化：启用 KV-Cache
- 批处理优化：调整 batch size
- 量化优化：使用 INT4/INT8 量化

## KV-Cache 推理缓存

### 什么是 KV-Cache？

KV-Cache 是一种推理加速技术，通过缓存 Key-Value 对象来加速生成式模型的推理过程。

### 启用 KV-Cache

1. 进入"系统管理" → "系统设置" → "模型管理"
2. 找到"推理缓存 (KV-Cache)"开关
3. 开启 KV-Cache
4. 保存设置

### 效果

- 推理速度提升 30-50%
- 显存占用增加 10-20%
- 特别适合长文本生成场景

## 模型负载均衡

### 1. 多副本部署

部署同一模型的多个副本到不同 GPU：

**步骤**：
1. 上传模型时，选择"多副本部署"
2. 选择副本数量（1-4）
3. 系统自动分配到不同 GPU
4. 请求自动路由到空闲副本

### 2. 负载均衡策略

进入"系统管理" → "系统设置" → "集群配置"，配置：
- **轮询（Round Robin）**：请求依次分配
- **最少连接（Least Connections）**：分配到连接数最少的副本
- **加权轮询（Weighted RR）**：按权重比例分配

## 常见问题

**Q: 模型部署失败，提示显存不足？**
A: 尝试以下方案：
1. 卸载其他不使用的模型释放显存
2. 使用量化版本（INT4/INT8）
3. 分割模型到多个 GPU（模型并行）

**Q: 模型推理速度慢？**
A: 优化方案：
1. 启用 KV-Cache
2. 增加 batch size
3. 使用量化版本
4. 升级到更快的 GPU（H100）

**Q: 如何评估模型性能？**
A: 查看模型详情的性能分析图表，关注：
- 吞吐量（tokens/秒）
- P95 延迟
- 错误率

**Q: 模型更新后旧版本还能用吗？**
A: 默认情况下，模型更新会替换旧版本。如需保留旧版本，请使用版本管理功能。
