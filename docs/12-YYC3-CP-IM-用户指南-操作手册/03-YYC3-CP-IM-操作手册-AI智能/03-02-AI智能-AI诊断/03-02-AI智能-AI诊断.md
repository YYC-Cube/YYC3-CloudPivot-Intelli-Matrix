# YYC³-CP-IM-用户指南-AI智能-AI诊断

## 功能概述

AI诊断提供智能化的系统健康诊断能力，自动检测异常、分析根因、提供修复方案。

---

## 诊断类型

### 1. 节点诊断
- 硬件健康检查
- 操作系统健康
- 网络连通性
- 资源使用分析

### 2. GPU诊断
- GPU 温度异常
- GPU 利用率异常
- 显存使用异常
- GPU 错误日志

### 3. 模型诊断
- 模型性能评估
- 推理延迟分析
- 吞吐量分析
- 准确率评估

### 4. 网络诊断
- 网络延迟
- 带宽利用率
- 丢包率
- 连接数统计

---

## 操作步骤

### 执行全面诊断

1. 进入"AI诊断"页面
2. 点击"开始全面诊断"按钮
3. 选择诊断范围：
   - 全系统
   - 指定节点
   - 指定服务
4. 点击"确认"开始诊断
5. 等待诊断完成（通常 3-5 分钟）

### 查看诊断结果

1. 诊断完成后，显示诊断报告
2. 报告包含：
   - 系统健康评分（0-100）
   - 发现的问题列表
   - 问题严重程度（高/中/低）
   - 根因分析
   - 修复建议

### 一键修复

1. 在诊断报告中，找到可自动修复的问题
2. 点击"一键修复"按钮
3. 确认修复操作
4. 等待修复完成
5. 查看修复结果

### 查看修复历史

1. 点击页面右上角"修复历史"
2. 查看所有修复记录
3. 查看修复效果

## 诊断规则

### 内置诊断规则

系统内置数百条诊断规则，覆盖：

| 类别 | 规则数量 | 示例 |
|------|---------|------|
| 硬件 | 50+ | CPU 温度 > 80°C |
| GPU | 80+ | GPU 使用率 > 95% |
| 网络 | 40+ | 网络延迟 > 100ms |
| 存储 | 30+ | 磁盘 I/O 等待时间 > 100ms |
| 应用 | 100+ | 服务响应时间 > 5s |

### 自定义诊断规则

1. 进入"系统管理" → "系统设置" → "AI/LLM配置"
2. 点击"自定义规则"
3. 创建新规则：
   - 规则名称
   - 检测条件
   - 严重程度
   - 修复建议
4. 保存规则

## 定时诊断

### 配置定时诊断

1. 进入"系统管理" → "系统设置" → "智能告警"
2. 配置诊断 Cron 表达式
3. 默认：每天凌晨 3:00

### 诊断通知

1. 进入"系统管理" → "系统设置" → "通知配置"
2. 配置通知渠道：
   - 邮件通知
   - Slack 通知
   - Webhook 通知
3. 设置告警阈值：
   - 高危问题：立即通知
   - 中危问题：每小时通知
   - 低危问题：每天汇总

## 常见问题

**Q: 诊断耗时太长怎么办？**
A: 可以选择诊断范围，只诊断问题节点，或调整诊断深度

**Q: 诊断结果不准确？**
A: 提供更多系统上下文，或更新诊断规则

**Q: 一键修复失败了怎么办？**
A: 查看修复日志，了解失败原因，然后手动修复

**Q: 如何导出诊断报告？**
A: 在诊断报告页面，点击"导出 PDF"或"导出 Excel"按钮

## 诊断示例

### 示例 1：GPU 温度过高

**问题描述**：GPU-03 节点温度 92°C

**诊断结果**：
- 严重程度：高
- 根因分析：散热风扇转速过低（1200 RPM），GPU 风扇未满速运行
- 修复建议：
  1. 检查风扇控制策略
  2. 手动提高风扇转速到 2500 RPM
  3. 检查散热器是否堵塞

**一键修复**：提高风扇转速到 2500 RPM

### 示例 2：网络延迟过高

**问题描述**：节点间网络延迟 150ms

**诊断结果**：
- 严重程度：中
- 根因分析：交换机端口拥塞，网络带宽利用率 98%
- 修复建议：
  1. 检查网络流量来源
  2. 调整 QoS 策略
  3. 考虑升级网络带宽
