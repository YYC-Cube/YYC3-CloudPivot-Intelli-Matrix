# YYC³-CP-IM-用户指南-运维管理-操作中心

## 功能概述

操作中心是执行各类运维操作的核心控制台，支持节点操作、模型操作、任务操作、系统操作和自定义操作。

---

## 操作类型

### 1. 全部操作
- 查看所有操作记录
- 按时间/类型/状态筛选
- 批量操作历史查询

### 2. 节点操作
- 重启节点
- 停止节点
- 节点维护模式
- 节点清理缓存

### 3. 模型操作
- 部署新模型
- 卸载模型
- 模型热更新
- 模型配置调整

### 4. 任务操作
- 创建新任务
- 暂停/恢复任务
- 终止任务
- 任务优先级调整

### 5. 系统操作
- 系统重启
- 服务重启
- 配置重载
- 日志清理

### 6. 自定义操作
- 创建自定义操作脚本
- 参数化操作
- 操作模板管理
- 批量执行

---

## 操作步骤

### 执行节点操作

1. 进入"操作中心" → "节点操作"
2. 选择目标节点
3. 选择操作类型
4. 填写参数（如有）
5. 点击"执行"按钮
6. 查看操作结果

### 执行模型操作

1. 进入"操作中心" → "模型操作"
2. 选择目标模型
3. 选择操作类型
4. 上传模型文件（部署时）
5. 点击"执行"按钮

### 创建自定义操作

1. 进入"操作中心" → "自定义"
2. 点击"新建操作"
3. 填写操作信息：
   - 操作名称
   - 操作类型
   - 执行脚本
   - 参数定义
4. 点击"保存"

---

## 操作状态

- **待执行**：操作已创建，等待执行
- **执行中**：操作正在执行
- **已完成**：操作成功完成
- **已失败**：操作执行失败
- **已取消**：操作被取消

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `N` | 节点操作 |
| `M` | 模型操作 |
| `T` | 任务操作 |
| `S` | 系统操作 |

---

## 常见问题

**Q: 操作失败如何重试？**
A: 在操作历史中找到失败的操作，点击"重试"按钮

**Q: 如何取消正在执行的操作？**
A: 在操作列表中找到执行中的操作，点击"取消"按钮

**Q: 批量操作支持吗？**
A: 支持。勾选多个节点/模型/任务，点击"批量操作"按钮
