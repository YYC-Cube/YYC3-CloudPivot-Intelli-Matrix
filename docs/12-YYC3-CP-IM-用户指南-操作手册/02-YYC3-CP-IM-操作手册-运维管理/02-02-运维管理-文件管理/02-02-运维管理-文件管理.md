## 功能概述

文件管理是 YYC³ CP-IM 的本地文件系统管理工具，提供文件浏览、日志查看、报告生成等核心功能，支持对宿主机存储的自动识别和可编辑管理。

### 核心能力

- **文件浏览**：浏览和管理本地文件系统
- **日志查看**：实时查看和筛选系统日志
- **报告生成**：生成各类运维报告
- **快速操作**：一键下载日志、执行备份、清理缓存
- **多标签页**：文件、日志、报告三个独立标签页
- **智能搜索**：支持文件和日志的快速搜索
- **格式化显示**：自动格式化文件大小和日期

## 操作步骤

### 1. 浏览文件

**步骤**：
1. 进入"运维管理" → "文件管理"
2. 默认显示"文件浏览"标签页
3. 文件浏览器显示当前目录的内容
4. 点击文件夹进入子目录
5. 点击面包屑导航返回上级目录

**文件类型**：
- 📁 **文件夹**：可点击进入
- 📄 **文件**：可点击查看详情

### 2. 查看文件详情

**步骤**：
1. 在文件浏览器中，点击任意文件
2. 右侧抽屉面板滑出，显示文件详情

**文件详情包含**：
- 文件名
- 文件类型
- 文件大小
- 修改时间
- 文件路径
- 文件内容（文本文件）

### 3. 搜索文件

**步骤**：
1. 在文件浏览器顶部，点击搜索框
2. 输入文件名或关键词
3. 系统实时过滤显示匹配的文件

### 4. 导航文件系统

**步骤**：
1. **进入文件夹**：点击文件夹图标
2. **返回上级**：点击面包屑导航中的上级目录
3. **快速跳转**：点击面包屑导航中的任意目录

**面包屑导航示例**：
```
~ / .yyc3-cloudpivot / configs / system
```

### 5. 查看日志

**步骤**：
1. 点击顶部"日志查看"标签页
2. 日志查看器显示所有系统日志
3. 使用筛选器过滤日志

**日志筛选**：
- **按级别**：全部/调试/信息/警告/错误
- **按来源**：全部/系统/应用/网络/数据库
- **搜索**：输入关键词搜索日志内容

**日志级别说明**：
- **调试**：详细的调试信息
- **信息**：常规操作信息
- **警告**：潜在问题警告
- **错误**：错误信息

### 6. 生成报告

**步骤**：
1. 点击顶部"报告生成"标签页
2. 报告生成器显示可生成的报告类型
3. 选择报告类型：
   - 系统健康报告
   - 性能分析报告
   - 巡查历史报告
   - 告警统计报告
4. 点击"生成报告"按钮
5. 等待报告生成完成
6. 下载或查看报告

### 7. 快速操作

#### 7.1 下载日志

**步骤**：
1. 点击快速操作栏中的"下载日志"按钮
2. 系统自动打包所有日志文件
3. 下载到本地（ZIP 格式）

#### 7.2 执行备份

**步骤**：
1. 点击快速操作栏中的"执行备份"按钮
2. 系统自动备份配置文件
3. 备份文件保存到 `backups/` 目录
4. 备份文件命名格式：`backup-YYYYMMDD-HHMMSS.tar.gz`

#### 7.3 清理缓存

**步骤**：
1. 点击快速操作栏中的"清理缓存"按钮
2. 系统自动清理缓存文件
3. 释放磁盘空间
4. 显示清理结果

## 常见问题

### Q1: 无法访问某些文件？

**A**:
1. 检查文件权限
2. 确认当前用户有访问权限
3. 使用 `sudo` 权限启动应用（如需要）

### Q2: 日志文件过大怎么办？

**A**:
1. 使用日志筛选器，只查看需要的日志
2. 定期执行"清理缓存"操作
3. 配置日志轮转策略（在系统设置中）

### Q3: 备份文件保存在哪里？

**A**:
备份文件默认保存在 `~/.yyc3-cloudpivot/backups/` 目录下，按类型分类：
- `configs/`：配置文件备份
- `data/`：数据文件备份

### Q4: 如何恢复备份？

**A**:
1. 进入文件管理器
2. 导航到 `backups/` 目录
3. 找到需要恢复的备份文件
4. 解压备份文件
5. 将文件复制到相应目录

### Q5: 报告生成失败怎么办？

**A**:
1. 检查磁盘空间是否充足
2. 检查日志文件是否完整
3. 查看系统日志，了解失败原因
4. 重新尝试生成报告

### Q6: 如何删除文件？

**A**:
1. 在文件浏览器中，点击文件查看详情
2. 在详情抽屉中，点击"删除"按钮
3. 确认删除操作

**注意**：删除操作不可逆，请谨慎操作

## 最佳实践

### 文件管理

- 定期清理缓存和临时文件
- 定期备份配置文件
- 使用有意义的文件名和目录名
- 避免在系统目录中存放个人文件

### 日志管理

- 定期查看日志，及时发现潜在问题
- 使用日志筛选器，提高查看效率
- 配置日志轮转，避免日志文件过大
- 保存重要日志，用于问题追溯

### 备份管理

- 定期执行备份（建议每天一次）
- 保留多个版本的备份
- 将备份文件存储到安全位置
- 定期测试备份恢复流程
