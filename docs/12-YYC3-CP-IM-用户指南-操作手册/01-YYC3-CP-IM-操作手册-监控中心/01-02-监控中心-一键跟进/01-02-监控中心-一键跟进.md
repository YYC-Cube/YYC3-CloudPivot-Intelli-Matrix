## 功能概述

一键跟进是 YYC³ CP-IM 的智能告警处理系统，提供异常任务的快速处理流程，支持批量操作、自动化处理和智能分析。

### 核心能力

- **智能告警聚合**：自动聚合相关告警，形成事件链
- **快速处理**：一键修复、标记解决、批量操作
- **智能分析**：AI 辅助分析告警根因
- **事件追踪**：完整的事件链路追踪
- **优先级管理**：按严重程度自动排序（严重/错误/警告/信息）
- **多维度筛选**：按严重程度、状态、标签筛选

## 操作步骤

### 1. 查看待处理告警

**步骤**：
1. 登录系统后，进入"监控中心" → "一键跟进"
2. 主面板显示所有需要关注的告警
3. 告警按优先级自动排序（严重 > 错误 > 警告 > 信息）
4. 顶部统计卡片显示各严重程度的告警数量

**说明**：
- **严重**：需要立即处理，影响系统核心功能
- **错误**：需要尽快处理，影响部分功能
- **警告**：需要关注，可能影响性能
- **信息**：系统通知，无需处理

### 2. 查看告警详情

**步骤**：
1. 点击告警卡片上的"查看详情"按钮
2. 右侧抽屉面板滑出，显示完整信息

**告警详情包含**：
- 告警标题和严重程度
- 来源节点/服务
- 指标值和阈值
- 当前状态
- 触发时间
- 负责人
- 标签
- 相关告警
- **事件链**：完整的事件追踪链路

**事件链示例**：
```
┌─────────────────────────────────────────────────────────┐
│ 事件链                                                  │
├─────────────────────────────────────────────────────────┤
│ 10:23:45  模型加载 LLaMA-70B                           │
│          节点: GPU-A100-03 · 显存分配 68GB             │
│                                                          │
│ 10:24:12  推理任务启动                                  │
│          任务: #12847 · Batch Size: 32                 │
│                                                          │
│ 10:24:15  延迟异常告警 ← 当前                          │
│          告警: #AL-0032 · 延迟 2,450ms > 阈值 2,000ms  │
│                                                          │
│ 10:24:30  系统自动降频                                  │
│          操作: auto_scale_down · 降低 Batch Size 至 16 │
└─────────────────────────────────────────────────────────┘
```

### 3. 一键修复

**步骤**：
1. 在告警卡片上，点击"一键修复"按钮
2. 系统自动执行预定义的修复策略
3. 告警状态自动更新为"分析中"
4. 系统显示修复进度和结果

**支持的自动修复**：
- GPU 延迟异常：自动降低 Batch Size
- 显存不足：自动清理缓存、拒绝新任务
- 存储空间不足：自动清理临时文件
- 网络延迟：自动路由优化

### 4. 标记为已解决

**步骤**：
1. 在告警卡片上，点击"标记解决"按钮
2. 或在详情抽屉中，点击"标记为已解决"
3. 告警状态更新为"已解决"
4. 告警从活跃列表中移除

### 5. 批量处理

**步骤**：
1. 勾选多个告警卡片
2. 点击顶部"批量操作"按钮
3. 选择统一处理方式：
   - 批量一键修复
   - 批量标记解决
   - 批量分配给用户
4. 确认执行

### 6. 筛选告警

**步骤**：
1. 使用顶部筛选器进行筛选
2. **按严重程度**：
   - 全部
   - 严重
   - 错误
   - 警告
   - 信息
3. **按状态**：
   - 全部
   - 活跃
   - 分析中
   - 已解决
   - 忽略

### 7. 查看处理历史

**步骤**：
1. 在状态筛选器中，选择"已解决"
2. 查看所有已解决的告警
3. 点击"查看详情"查看处理过程

## 常见问题

### Q1: 一键修复失败怎么办？

**A**:
1. 查看告警详情中的事件链，了解失败原因
2. 手动执行修复步骤
3. 联系系统管理员
4. 在"AI 智能" → "AI 决策"中咨询解决方案

### Q2: 如何取消正在执行的修复？

**A**:
1. 在告警列表中找到"分析中"状态的告警
2. 点击"取消"按钮
3. 系统会停止自动修复，恢复到之前状态

### Q3: 告警过多无法处理？

**A**:
1. 使用筛选器按严重程度筛选，优先处理严重告警
2. 使用批量操作功能，批量修复或标记解决
3. 启用 AI 自动处理，系统会自动处理低优先级告警

### Q4: 如何导出告警数据？

**A**:
1. 在一键跟进页面，点击右上角"导出"按钮
2. 选择导出格式（CSV/JSON/PDF）
3. 选择时间范围和筛选条件
4. 点击确认下载

### Q5: 事件链中的"自动操作"是什么？

**A**:
事件链中的"自动操作"是系统根据告警类型自动执行的修复策略，例如：
- 自动降频：降低 GPU 频率或 Batch Size
- 自动清理：清理缓存或临时文件
- 自动拒绝：拒绝新任务以释放资源
- 自动切换：切换到备用节点或模型

💡 **提示 1**：开启"自动修复"后，系统会自动处理常见告警，减少人工干预

## 最佳实践

### 告警处理优先级

1. **严重告警**：立即处理，影响核心功能
2. **错误告警**：尽快处理，影响部分功能
3. **警告告警**：关注处理，可能影响性能
4. **信息告警**：了解即可，无需处理

### 自动修复策略

- 对于常见问题，优先使用一键修复
- 对于复杂问题，先查看事件链，了解根因
- 对于未知问题，咨询 AI 决策或联系管理员

### 批量操作

- 相似类型的告警可以批量处理
- 批量操作前，先查看告警详情，确认处理方式
- 批量修复时，注意系统负载，避免同时处理过多
