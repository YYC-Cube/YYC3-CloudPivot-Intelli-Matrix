## 功能概述

巡查模式是 YYC³ CP-IM 的自动化系统巡检功能，提供周期性、全面的系统健康检查，生成详细的巡检报告和修复建议。

### 核心能力

- **自动化巡检**：一键启动全系统巡检，自动检查所有关键指标
- **实时进度**：显示巡检进度和当前检查项
- **智能评分**：基于检查结果计算系统健康评分（0-100）
- **详细报告**：生成包含问题列表和修复建议的巡检报告
- **定时巡查**：支持定时自动巡查，无需人工干预
- **历史记录**：保存所有巡查历史，支持追溯和对比
- **问题分类**：按严重程度分类问题（通过/警告/严重/跳过）

## 操作步骤

### 1. 启动手动巡查

**步骤**：
1. 进入"监控中心" → "巡查模式"
2. 点击右上角"手动巡查"按钮
3. 系统开始执行全系统巡检
4. 进度条显示当前巡查进度（0-100%）

**巡查过程**：
- 系统按顺序检查所有项目
- 每个检查项耗时约 150-250ms
- 总耗时约 8-15 秒（20 个检查项）
- 巡查过程中可查看实时进度

### 2. 查看巡查进度

**步骤**：
1. 巡查启动后，主面板显示进度条
2. 显示当前进度百分比
3. 显示正在检查的项目

**进度条示例**：
```

│ ████████████████████████░░░░░░░░░░░░░░░░░░░░░░░░░░░░   │
└─────────────────────────────────────────────────────────┘
```

### 3. 查看巡查结果

**步骤**：
1. 巡查完成后，自动显示当前结果
2. 主面板显示健康评分和检查统计
3. 详细检查项列表显示在下方

**结果统计**：
- **健康评分**：系统整体健康度（0-100）
- **通过**：检查通过的项目数量
- **警告**：有警告的项目数量
- **严重**：严重问题的项目数量

**检查项状态**：
- ✅ **通过**：指标正常，无需处理
- ⚠️ **警告**：指标异常，需要关注
- 🔴 **严重**：严重问题，需要立即处理
- ⏭️ **跳过**：检查项跳过（如节点离线）

### 4. 查看详细报告

**步骤**：
1. 在巡查历史中，点击"查看"按钮
2. 弹出详细报告模态框
3. 报告包含：
   - 巡查概要（时间、耗时、健康评分）
   - 检查统计（总数、通过、警告、严重、跳过）
   - 详细检查项列表（按类别分组）

### 5. 配置定时巡查

**步骤**：
1. 点击右上角"⚙️ 巡查计划"按钮
2. 巡查计划面板展开
3. 配置以下选项：
   - **启用自动巡查**：开启/关闭定时巡查
   - **巡查间隔**：5/10/15/30/60 分钟
4. 保存设置

**巡查计划面板**：

### 6. 查看巡查历史

**步骤**：
1. 主面板下方显示巡查历史列表
2. 每条记录包含：
   - 巡查时间
   - 触发方式（手动/自动/定时）
   - 健康评分
   - 检查统计
   - 查看按钮
3. 点击"查看"按钮查看详细报告

**步骤**：
1. 在巡查报告模态框中，点击"导出"按钮
2. 选择导出格式（PDF/Excel/Markdown）
3. 确认下载

## 巡查项目详解

### 1. 节点健康检查

**检查项**：
- GPU 节点在线状态
- GPU 使用率
- 显存使用率
- 节点运行时长
- 推理延迟

**阈值**：
- GPU 使用率：< 90%
- 显存使用率：< 95%
- 推理延迟：< 2,000ms

### 2. 存储检查

**检查项**：
- NAS 存储使用率
- 本地 SSD 缓存使用率
- 数据库存储使用率

**阈值**：
- NAS 存储使用率：< 85%
- 本地 SSD 缓存使用率：< 90%
- 数据库存储使用率：< 80%

### 3. 网络检查

**检查项**：
- 节点互联延迟
- 外部网络延迟
- WebSocket 连接状态

**阈值**：
- 节点互联延迟：< 30ms
- 外部网络延迟：< 50ms

### 4. 模型服务检查

**检查项**：
- 模型服务状态
- 模型推理延迟
- 模型加载状态

**阈值**：
- 模型推理延迟：< 1,500ms

### 5. 数据库检查

**检查项**：
- 数据库连接状态
- 数据库存储使用率
- 活动连接数

**阈值**：
- 数据库存储使用率：< 80%
- 活动连接数：< 100

### 6. 进程检查

**检查项**：
- 推理引擎进程
- API 网关进程
- 日志收集器进程

**阈值**：
- 所有关键进程必须运行

## 健康评分计算

健康评分基于所有检查项的加权计算：

| 状态 | 权重 | 说明 |
|------|------|------|
| **通过** | 1.0 | 完全正常 |
| **警告** | 0.6 | 有异常，但可接受 |
| **严重** | 0.0 | 严重问题，需立即处理 |
| **跳过** | 0.5 | 节点离线，不计入评分 |

**计算公式**：
```
健康评分 = (通过数 × 1.0 + 警告数 × 0.6 + 跳过数 × 0.5) / 总检查数 × 100
```

**评分等级**：
- **90-100**：优秀（绿色）
- **80-89**：良好（青色）
- **70-79**：一般（黄色）
- **60-69**：较差（橙色）
- **<60**：严重（红色）

## 常见问题

### Q1: 巡查中断了怎么办？

**A**:
1. 巡查会自动保存当前进度
2. 可以在巡查页面点击"继续巡查"，从上次中断处继续
3. 如果中断时间过长，建议重新启动巡查

### Q2: 如何修改巡查间隔？

**A**:
1. 点击右上角"⚙️ 巡查计划"按钮
2. 在"巡查间隔"下拉菜单中选择新的间隔
3. 保存设置后立即生效

### Q3: 巡查耗时过长怎么办？

**A**:
1. 检查网络连接，确保节点间通信正常
2. 检查是否有节点离线，导致超时
3. 在系统设置中调整巡查超时时间

### Q4: 如何导出巡查报告？

**A**:
1. 在巡查历史中，点击"查看"按钮打开报告
2. 在报告模态框中，点击"导出"按钮
3. 选择导出格式（PDF/Excel/Markdown）
4. 确认下载

### Q5: 健康评分突然下降怎么办？

**A**:
1. 查看详细报告，找出问题项
2. 优先处理严重问题（红色标记）
3. 处理警告问题（黄色标记）
4. 重新执行巡查，验证修复效果

### Q6: 如何禁用自动巡查？

**A**:
1. 点击右上角"⚙️ 巡查计划"按钮
2. 取消勾选"启用自动巡查"
3. 保存设置

### Q7: 巡查发现严重问题后，系统会自动处理吗？

**A**:
巡查模式仅负责检查和报告，不会自动处理。如需自动处理，请使用"一键跟进"功能，或配置自动化规则。

## 最佳实践

### 巡查频率建议

- **生产环境**：每 15-30 分钟自动巡查 + 每天手动巡查
- **测试环境**：每 30-60 分钟自动巡查 + 每周手动巡查
- **开发环境**：每小时自动巡查 + 需要时手动巡查

### 问题处理优先级

1. **严重问题**：立即处理，影响系统核心功能
2. **警告问题**：尽快处理，可能影响性能
3. **通过项**：无需处理，保持监控

### 巡查报告管理

- 定期导出巡查报告，用于趋势分析
- 保存重要巡查报告，用于问题追溯
- 分享巡查报告给团队，提高问题意识
