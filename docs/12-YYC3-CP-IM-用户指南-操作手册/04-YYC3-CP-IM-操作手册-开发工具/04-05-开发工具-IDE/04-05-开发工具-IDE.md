## 第一章：功能概述 —— 智能编程的核心能力

### 1.1 IDE 核心定位

YYC³ CP-IM 的 IDE 集成是一个**功能完整的在线开发环境**，旨在为开发者提供一站式的代码编辑、调试、版本控制和 AI 辅助体验。

### 1.2 核心特性矩阵

| 特性类别 | 功能描述 | 技术实现 |
|---------|---------|---------|
| **代码编辑** | 语法高亮、代码补全、代码格式化、多光标编辑 | Monaco Editor |
| **调试支持** | 断点设置、变量查看、调用栈、单步执行 | Chrome DevTools Protocol |
| **版本控制** | Git 集成、分支管理、提交/推送、冲突解决 | isomorphic-git |
| **AI 辅助** | 代码生成、代码优化、代码解释、错误修复 | 集成现有 AI 功能 |
| **终端集成** | 多终端支持、命令历史、Shell 集成 | xterm.js |
| **文件管理** | 文件树、文件操作、搜索过滤 | Electron API |
| **多标签页** | 同时编辑多个文件、标签页切换、未保存提示 | React State |
| **扩展生态** | 插件市场、自定义扩展、快捷键配置 | 扩展系统 |

- 📤 **交互增强**：支持拖拽文件到终端、终端输出可复制和搜索

**技术实现**：
```typescript
// 使用 xterm.js
import { Terminal } from 'xterm';
import { FitAddon } from 'xterm-addon-fit';
import { WebLinksAddon } from 'xterm-addon-web-links';

function IntegratedTerminal() {
  const terminalRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    const terminal = new Terminal({
      cursorBlink: true,
      fontSize: 14,
      fontFamily: "'JetBrains Mono', monospace",
    });

    const fitAddon = new FitAddon();
    const webLinksAddon = new WebLinksAddon();

    terminal.loadAddon(fitAddon);
    terminal.loadAddon(webLinksAddon);

    terminal.open(terminalRef.current);
    fitAddon.fit();

    return () => terminal.dispose();
  }, []);

  return <div ref={terminalRef} className="terminal-container" />;
}
```

## 第四章：操作指南 —— 从入门到精通

### 4.1 快速入门

#### 4.1.1 打开 IDE

**方法一：快捷键**
- 按 `Ctrl + Shift + I` 快捷键

**方法二：顶部状态栏**
- 点击顶部状态栏的 IDE 图标

**方法三：导航菜单**
- 进入"开发工具" → "IDE"

#### 4.1.2 打开项目

**步骤**：
1. 在 IDE 中，点击"打开文件夹"按钮
2. 选择项目目录
3. 等待项目加载完成
4. 文件树将显示项目结构

#### 4.1.3 编辑代码

**步骤**：
1. 在文件树中找到要编辑的文件
2. 双击打开文件
3. 开始编辑代码
4. 按 `Ctrl + S` 保存文件
5. 未保存的文件将显示 `●` 标记

---

### 4.2 AI 辅助使用

#### 4.2.1 AI 代码生成

**步骤**：
1. 在左栏的 AI 交互区域，输入自然语言描述
2. 例如："创建一个 React 组件，显示用户列表"
3. AI 将自动生成代码
4. 点击"插入代码"将代码插入到编辑器

#### 4.2.2 AI 代码优化

**步骤**：
1. 在编辑器中选中要优化的代码
2. 点击"优化代码"按钮
3. AI 将分析代码并提供优化建议
4. 点击"应用优化"应用优化后的代码

#### 4.2.3 AI 错误修复

**步骤**：
1. 当代码出现错误时，点击"修复错误"按钮
2. AI 将分析错误并提供修复建议
3. 点击"应用修复"应用修复后的代码

---

### 4.3 终端使用

#### 4.3.1 打开终端

**步骤**：
1. 按 `Ctrl + `` ` (反引号) 快捷键
2. 或点击右栏的终端图标
3. 终端将在右栏底部打开

#### 4.3.2 执行命令

**步骤**：
1. 在终端中输入命令
2. 按 `Enter` 执行命令
3. 终端将显示命令输出

#### 4.3.3 多终端管理

**步骤**：
1. 点击终端标签页的 `+` 按钮
2. 创建新的终端实例
3. 点击终端标签页切换不同终端
4. 点击 `×` 关闭终端

---

### 4.4 Git 版本控制

#### 4.4.1 查看修改

**步骤**：
1. 按 `Ctrl + Shift + G` 打开 Git 面板
2. 查看修改的文件列表
3. 点击文件查看具体修改内容

#### 4.4.2 提交代码

**步骤**：
1. 在 Git 面板中，选择要提交的文件
2. 输入提交信息
3. 点击"提交"按钮
4. 代码将提交到本地仓库

#### 4.4.3 推送到远程

**步骤**：
1. 在 Git 面板中，点击"推送"按钮
2. 代码将推送到远程仓库
3. 等待推送完成

#### 4.4.4 切换分支

**步骤**：
1. 点击左下角分支名称
2. 选择要切换的分支
3. 点击"切换"按钮
4. 等待分支切换完成

---

### 4.5 调试代码

#### 4.5.1 设置断点

**步骤**：
1. 在代码行号左侧点击
2. 红色圆点表示断点已设置
3. 可以设置多个断点

#### 4.5.2 开始调试

**步骤**：
1. 按 `F5` 或点击"开始调试"按钮
2. 程序将在断点处暂停
3. 查看变量值、调用栈

#### 4.5.3 单步执行

**步骤**：
1. 按 `F10` 单步执行（不进入函数）
2. 按 `F11` 进入函数
3. 按 `Shift + F11` 跳出函数
4. 按 `F9` 继续执行
