**受影响的组件：**
- ErrorBoundary
- ToastContainer
- PWAInstallPrompt
- OfflineIndicator  
- CommandPalette
- IntegratedTerminal
- 等...

**预计修复时间：** 30-45 分钟

**修复方法：**

```typescript
// 错误的写法
vi.mock("../components/ErrorBoundary", () => ({
  ErrorBoundary: (...) => {...}
}));

// 正确的写法
vi.mock("../components/ErrorBoundary", () => ({
  default: (...) => {...}
}));
```

## 💡 关键教训

1. **版本控制至关重要**
   - 已初始化 Git 仓库
   - 每次修改前创建分支
   - 小步提交，频繁验证

2. **优化需要渐进式实施**
   - 先实施无风险的优化（配置类）
   - 再实施有风险的优化（代码类）
   - 每步都要验证测试

3. **Mock 语句要统一格式**
   - 所有组件 mock 都要用默认导出
   - 建立统一的 mock 规范
   - 使用自动化工具检查
