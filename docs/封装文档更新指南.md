# 封装文档更新指南

## 概述

本文档提供项目代码更新后，封装文档同步更新的详细指南，确保文档始终与项目实际保持一致。

---

## 一、自动化检查工具

### 1.1 运行检查脚本

项目提供了自动化检查脚本，可以快速检测文档是否需要更新：

```bash
# 运行文档同步检查
./scripts/check-docs-sync.sh
```

**检查内容：**

| 检查项 | 说明 |
|---------|------|
| 版本号 | 检查 package.json 版本与文档版本是否一致 |
| 依赖版本 | 检查 Electron、React、Vite 版本是否需要更新 |
| 项目结构 | 检查关键目录是否存在且文档已记录 |
| 构建配置 | 检查端口配置和构建输出目录是否匹配 |

**输出示例：**

```
🔍 检查封装文档更新需求...
================================
📋 检查 1: 版本号
----------------------------------------
✅ 版本号一致

📋 检查 2: 依赖版本
----------------------------------------
   当前依赖版本:
   - Electron: ^28.0.0
   - React: ^19.2.4
   - Vite: ^7.3.1
   ⚠️  Electron 版本可能需要更新
   ⚠️  React 版本可能需要更新
   ⚠️  Vite 版本可能需要更新
```

## 二、手动更新指南

### 2.1 版本号更新

**触发条件：** 每次发布新版本

**步骤：**

1. **更新 package.json 版本号**

   ```json
   {
     "version": "1.0.1"
   }
   ```

2. **更新文档中的版本号**

   需要更新的文档：
   - `docs/03-YYC3-CP-IM-项目开发-实施阶段/0306-CP-IM-集成指南/014-CP-IM-开发实施阶段-桌面应用最终封装报告.md`

   搜索并替换：
   ```bash
   # 查找所有版本号
   grep -n "版本.*[0-9]\+\.[0-9]\+\.[0-9]\+" docs/03-YYC3-CP-IM-项目开发-实施阶段/0306-CP-IM-集成指南/014-CP-IM-开发实施阶段-桌面应用最终封装报告.md

   # 替换版本号（示例）
   sed -i '' 's/1\.0\.0/1.0.1/g' docs/03-YYC3-CP-IM-项目开发-实施阶段/0306-CP-IM-集成指南/014-CP-IM-开发实施阶段-桌面应用最终封装报告.md
   ```

### 2.2 依赖版本更新

**触发条件：** 升级 Electron、React、Vite 等核心依赖

**步骤：**

1. **检查当前依赖版本**

   ```bash
   grep -E '"(electron|react|vite)":' package.json
   ```

2. **更新文档中的技术栈部分**

   需要更新的文档：
   - `docs/03-YYC3-CP-IM-项目开发-实施阶段/0306-CP-IM-集成指南/006-CP-IM-开发实施阶段-桌面应用封装操作指南.md`
   - `docs/03-YYC3-CP-IM-项目开发-实施阶段/0306-CP-IM-集成指南/014-CP-IM-开发实施阶段-桌面应用最终封装报告.md`

   更新位置：
   ```markdown
   ## 技术栈

   **前端框架**：
   - React 19.2.4  ← 更新此版本号
   - TypeScript 5.9.3
   - Vite 7.3.1  ← 更新此版本号

   **桌面框架**：
   - Electron 28.0.0  ← 更新此版本号
   - electron-builder 24.13.3
   ```

### 2.3 项目结构更新

**触发条件：** 添加/删除目录或重构项目结构

**步骤：**

1. **检查当前项目结构**

   ```bash
   tree -L 2 -I 'node_modules|dist|dist-electron|.git'
   ```

2. **更新文档中的项目结构部分**

   需要更新的文档：
   - `docs/03-YYC3-CP-IM-项目开发-实施阶段/0306-CP-IM-集成指南/006-CP-IM-开发实施阶段-桌面应用封装操作指南.md`

   更新位置：
   ```markdown
   ### 当前项目结构

   ```
   YYC³ CloudPivot Intelli-Matrix/
   ├── electron/                  # Electron 主进程代码
   │   ├── main.ts                # 主进程入口
   │   ├── preload.ts             # 预加载脚本
   │   └── tsconfig.json          # TypeScript 配置
   ├── src/                       # React 应用源码
   ├── dist/                      # Vite 构建输出（前端资源）
   ├── dist-electron/             # Electron 构建输出
   └── package.json               # 项目配置
   ```
   ```

   **注意：**
   - 添加新目录时，在文档中添加对应的说明
   - 删除目录时，从文档中移除对应的说明
   - 更新目录用途说明

### 2.4 构建配置更新

**触发条件：** 修改 vite.config.ts 或 electron-builder 配置

**步骤：**

1. **检查当前构建配置**

   ```bash
   # 检查 Vite 配置
   grep -A 5 "server:" vite.config.ts

   # 检查 electron-builder 配置
   grep -A 10 '"build":' package.json
   ```

2. **更新文档中的构建配置部分**

   需要更新的文档：
   - `docs/03-YYC3-CP-IM-项目开发-实施阶段/0306-CP-IM-集成指南/006-CP-IM-开发实施阶段-桌面应用封装操作指南.md`

   更新位置：
   ```markdown
   ### 1. 环境变量配置

   创建 `.env` 文件：

   ```env
   # 应用端口
   VITE_PORT=3218  ← 如果端口变化，更新此值
   ```
   ```

**触发条件：** 新增/删除功能或修改功能行为

**步骤：**

   # 查找所有组件
   ls src/app/components/
   ```

2. **更新文档中的功能测试清单**

   需要更新的文档：

   更新位置：
   ```markdown
   ### 2. 功能测试清单

   - [ ] 应用正常启动
   - [ ] 窗口尺寸正确（1400×900）
   - [ ] 系统托盘图标显示

## 三、更新检查清单

### 3.1 发布前检查

在每次发布新版本前，完成以下检查：

- [ ] 运行 `./scripts/check-docs-sync.sh` 检查文档同步状态
- [ ] 更新 package.json 版本号
- [ ] 更新所有文档中的版本号
- [ ] 检查依赖版本是否需要更新
- [ ] 检查项目结构是否需要更新
- [ ] 检查构建配置是否需要更新
- [ ] 检查功能清单是否需要更新
- [ ] 验证文档中的命令是否正确
- [ ] 验证文档中的路径是否正确
- [ ] 更新文档的 `@updated` 字段

### 3.2 依赖升级后检查

在升级核心依赖后，完成以下检查：

- [ ] 更新文档中的技术栈版本号
- [ ] 检查新版本的 API 变化
- [ ] 更新文档中的代码示例
- [ ] 测试文档中的命令是否仍然有效
- [ ] 更新文档的 `@updated` 字段

### 3.3 项目结构变化后检查

在项目结构变化后，完成以下检查：

- [ ] 更新文档中的项目结构图
- [ ] 更新关键文件说明
- [ ] 更新目录用途说明
- [ ] 检查文档中的路径是否正确
- [ ] 更新文档的 `@updated` 字段

## 四、常见更新场景

### 4.1 场景 1：小版本更新（1.0.0 → 1.0.1）

**更新内容：**
- ✅ 版本号
- ✅ 更新日志

**不需要更新：**
- ❌ 技术栈版本
- ❌ 项目结构
- ❌ 构建配置

**更新步骤：**
```bash
# 1. 运行检查脚本
./scripts/check-docs-sync.sh

# 2. 更新版本号
sed -i '' 's/1\.0\.0/1.0.1/g' docs/03-YYC3-CP-IM-项目开发-实施阶段/0306-CP-IM-集成指南/014-CP-IM-开发实施阶段-桌面应用最终封装报告.md

# 3. 更新文档更新日期
sed -i '' 's/@updated: 2026-03-04/@updated: 2026-03-09/g' docs/03-YYC3-CP-IM-项目开发-实施阶段/0306-CP-IM-集成指南/*.md
```

### 4.2 场景 2：依赖升级（React 18 → 19）

**更新内容：**
- ✅ 技术栈版本
- ✅ 代码示例（如果 API 变化）
- ✅ 兼容性说明

**更新步骤：**
```bash
# 1. 运行检查脚本
./scripts/check-docs-sync.sh

# 2. 更新技术栈版本
sed -i '' 's/React 18\.3\.1/React 19.2.4/g' docs/03-YYC3-CP-IM-项目开发-实施阶段/0306-CP-IM-集成指南/014-CP-IM-开发实施阶段-桌面应用最终封装报告.md

# 3. 检查代码示例是否需要更新
# 4. 更新文档更新日期
```

### 4.3 场景 3：项目结构变化（添加新目录）

**更新内容：**
- ✅ 项目结构图
- ✅ 目录说明
- ✅ 文件说明

**更新步骤：**
```bash
# 1. 运行检查脚本
./scripts/check-docs-sync.sh

## 五、自动化更新脚本（可选）

### 5.1 创建版本更新脚本

```bash
#!/bin/bash

# 自动更新文档版本号
NEW_VERSION=$1

if [ -z "$NEW_VERSION" ]; then
    echo "用法: $0 <新版本号>"
    echo "示例: $0 1.0.1"
    exit 1
fi

echo "🔄 更新文档版本号到 $NEW_VERSION..."

# 更新所有文档中的版本号
find docs/03-YYC3-CP-IM-项目开发-实施阶段/0306-CP-IM-集成指南 -name "*.md" -exec sed -i '' "s/[0-9]\+\.[0-9]\+\.[0-9]\+/$NEW_VERSION/g" {} \;

# 更新文档更新日期
TODAY=$(date +%Y-%m-%d)
find docs/03-YYC3-CP-IM-项目开发-实施阶段/0306-CP-IM-集成指南 -name "*.md" -exec sed -i '' "s/@updated: [0-9]\{4\}-[0-9]\{2\}-[0-9]\{2\}/@updated: $TODAY/g" {} \;

echo "✅ 版本号更新完成！"
```

**使用方法：**
```bash
# 赋予执行权限
chmod +x scripts/update-docs-version.sh

# 更新版本号到 1.0.1
./scripts/update-docs-version.sh 1.0.1
```

## 六、最佳实践

### 6.1 定期检查

- **每周**：运行一次 `./scripts/check-docs-sync.sh`
- **每次发布前**：完成发布前检查清单
- **每次依赖升级后**：完成依赖升级后检查清单
- **每次项目结构变化后**：完成项目结构变化后检查清单

### 6.2 文档版本控制

- **提交文档更新**：每次更新文档后，立即提交到 Git
- **使用有意义的提交信息**：`docs: 更新封装文档版本号到 1.0.1`
- **创建文档版本标签**：每次发布时创建对应的文档版本标签

### 6.3 文档质量保证

- **验证文档中的命令**：确保所有命令可以正确执行
- **验证文档中的路径**：确保所有路径都正确
- **验证文档中的链接**：确保所有链接都有效
- **保持文档格式一致**：使用统一的格式和风格
