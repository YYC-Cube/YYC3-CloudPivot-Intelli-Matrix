## 📋 一、文档命名规范

### 1.1 标准命名格式

**格式**: `{序号}-{阶段}-{类型}-{主题}-{文档类型}.md`

**示例**:
- `001-CP-IM-项目设计阶段-物理硬件管理-架构设计.md`
- `0501-CP-IM-整合实施-文档重组计划.md`
- `0601-CP-IM-测试验证-阶段测试报告.md`

### 1.2 命名组成部分

| 组成部分 | 说明 | 格式 | 示例 |
|---------|------|------|------|
| 序号 | 三位数字,按顺序排列 | 001-999 | 001, 002, 0501 |
| 阶段 | 项目阶段缩写 | CP-IM | CP-IM (CloudPivot Intelli-Matrix) |
| 类型 | 文档类型 | 中文 | 项目设计阶段, 整合实施, 测试验证 |
| 主题 | 文档主题 | 中文 | 物理硬件管理, 文档重组 |
| 文档类型 | 文档类型 | 中文 | 架构设计, 执行报告, 计划 |

### 1.3 目录命名规范

**格式**: `{序号}-{阶段}-{类型}`

**示例**:
- `00-YYC³-CP-IM-项目总览索引`
- `01-YYC³-CP-IM-启动规划阶段`
- `02-YYC³-CP-IM-项目设计阶段`
- `05-YYC³-CP-IM-整合实施阶段`

## 📋 六、文档质量标准

### 6.1 文档头部格式

所有文档必须包含以下头部信息:

```markdown
---
@file: 文件名称.md
@description: 文档描述
@author: YanYuCloudCube Team
@version: 版本号
@created: 创建日期
@updated: 更新日期
@status: 状态
@tags: [标签1],[标签2]
---

> ***YanYuCloudCube***
> *言启象限 | 语枢未来*
> ***Words Initiate Quadrants, Language Serves as Core for Future***
> *万象归元于云枢 | 深栈智启新纪元*
> ***All things converge in cloud pivot; Deep stacks ignite a new era of intelligence***
```

### 6.2 文档状态定义

| 状态 | 说明 | 使用场景 |
|------|------|---------|
| `draft` | 草稿 | 正在编写的文档 |
| `proposed` | 提议 | 待审核的文档 |
| `review` | 审核中 | 正在审核的文档 |
| `approved` | 已批准 | 已批准的文档 |
| `published` | 已发布 | 已发布的文档 |
| `deprecated` | 已废弃 | 不再使用的文档 |
| `archived` | 已归档 | 归档的文档 |

### 6.3 文档质量检查清单

| 检查项 | 说明 | 状态 |
|--------|------|------|
| 命名规范 | 符合标准命名格式 | ✅ / ❌ |
| 头部完整 | 包含所有必要头部信息 | ✅ / ❌ |
| 结构清晰 | 章节结构清晰合理 | ✅ / ❌ |
| 内容准确 | 内容准确无误 | ✅ / ❌ |
| 格式统一 | 格式统一规范 | ✅ / ❌ |
| 链接有效 | 内部链接有效 | ✅ / ❌ |
| 无错别字 | 无错别字和语法错误 | ✅ / ❌ |
