## 版本策略

### 版本号规范

YYC³ CloudPivot Intelli-Matrix 遵循语义化版本控制（Semantic Versioning 2.0.0）：

```
主版本号.次版本号.修订号 (MAJOR.MINOR.PATCH)
```

| 版本号 | 说明 | 示例 |
|--------|------|------|
| **MAJOR** | 不兼容的 API 修改 | 1.0.0 → 2.0.0 |
| **MINOR** | 向下兼容的功能新增 | 1.0.0 → 1.1.0 |
| **PATCH** | 向下兼容的问题修正 | 1.0.0 → 1.0.1 |

### 预发布版本

| 标识 | 说明 | 示例 |
|------|------|------|
| **alpha** | 内部测试版本 | 1.0.0-alpha.1 |
| **beta** | 公开测试版本 | 1.0.0-beta.1 |
| **rc** | 候选发布版本 | 1.0.0-rc.1 |

### 发布分支策略

```
main (生产分支)
  ↑
  ↑ 合并
  ↑
develop (开发分支)
  ↑
  ↑ 合并
  ↑
feature/* (功能分支)
hotfix/* (热修复分支)
release/* (发布分支)
```

### 发布频率

| 发布类型 | 频率 | 说明 |
|---------|------|------|
| **主版本发布** | 按需 | 重大架构变更或 API 变更 |
| **次版本发布** | 每月 | 新功能发布 |
| **修订版本发布** | 每周 | Bug 修复和小改进 |
| **热修复发布** | 随时 | 紧急问题修复 |

## 发布流程

### 发布前准备

#### 1. 功能冻结

```bash
# 创建发布分支
git checkout develop
git pull origin develop
git checkout -b release/v1.1.0

# 更新版本号
# package.json
{
  "name": "yyc3-cloudpivot-intelli-matrix",
  "version": "1.1.0",
  ...
}

# electron/package.json
{
  "name": "yyc3-cloudpivot-intelli-matrix",
  "version": "1.1.0",
  ...
}
```

#### 2. 测试验证

```bash
# 运行所有测试
pnpm test

# 生成测试覆盖率报告
pnpm test:coverage

# 运行类型检查
pnpm type-check

# 运行代码检查
pnpm lint:fix

# 构建验证
pnpm build

# Electron 构建验证
pnpm build:electron
```

#### 3. 文档更新

```bash
# 更新 CHANGELOG.md
# 更新版本号
# 添加变更内容
# 更新发布日期

# 更新 README.md
# 更新版本信息
# 更新安装说明

# 更新 API 文档
# 更新组件文档
```

### 发布流程

#### 1. 代码审查

```bash
# 创建 Pull Request
# 从 release/v1.1.0 到 main

# 代码审查清单
- [ ] 代码符合团队规范
- [ ] 测试覆盖率达标
- [ ] 无类型错误
- [ ] 无 lint 警告
- [ ] 文档已更新
- [ ] 变更日志已更新
```

#### 2. 合并到主分支

```bash
# 合并 release 分支到 main
git checkout main
git merge --no-ff release/v1.1.0

# 创建版本标签
git tag -a v1.1.0 -m "Release version 1.1.0"

# 推送到远程仓库
git push origin main
git push origin v1.1.0
```

#### 3. 合并回开发分支

```bash
# 合并 main 到 develop
git checkout develop
git merge --no-ff main

# 推送到远程仓库
git push origin develop
```

#### 4. 构建发布

```bash
# 构建 Web 应用
pnpm build

# 构建 Electron 应用
pnpm build:electron

# 构建 Docker 镜像
pnpm docker:build
```

#### 5. 发布到平台

```bash
# 发布到 GitHub Releases
# 上传构建产物
# 发布 Docker 镜像
docker push yyc3/cloudpivot:1.1.0
docker push yyc3/cloudpivot:latest

# 发布 Electron 应用
# 上传到 GitHub Releases
# 发布到应用商店（可选）
```

## 版本回滚

### 回滚触发条件

| 条件 | 说明 | 处理方式 |
|------|------|----------|
| **严重 Bug** | 影响核心功能 | 立即回滚 |
| **性能问题** | 严重影响用户体验 | 评估后回滚 |
| **安全漏洞** | 存在安全风险 | 立即回滚 |
| **数据丢失** | 数据损坏或丢失 | 立即回滚 |

### 回滚流程

#### 1. 确认回滚

```bash
# 评估回滚影响
# 确认回滚范围
# 获得回滚批准
# 通知相关人员
```

#### 2. 执行回滚

```bash
# Web 应用回滚
git checkout v1.0.0
pnpm build
# 部署到生产环境

# Electron 应用回滚
# 发布旧版本到 GitHub Releases
# 通知用户更新

# Docker 回滚
docker pull yyc3/cloudpivot:1.0.0
docker stop yyc3-cpim-app
docker rm yyc3-cpim-app
docker run -d --name yyc3-cpim-app -p 3118:8080 yyc3/cloudpivot:1.0.0
```

#### 3. 数据库回滚

```bash
# 执行数据库回滚脚本
psql -h localhost -U postgres yyc3_cpim -f migrations/rollback_v1.1.0.sql

# 验证数据库状态
psql -h localhost -U postgres yyc3_cpim -c "SELECT version();"
```

#### 4. 验证回滚

```bash

# 验证监控告警
```
