# YYC³ 项目整合策略 v2

**策略原则**: 以 YYC³ CloudPivot Intelli-Matrix 为核心，渐进式整合前期版本功能
**目标**: 应用与宿主机存储一体，自动识别设备，支持写入操作，同步权限

---

## 📋 核心需求确认

### 1. 项目定位
- **核心项目**: YYC³ CloudPivot Intelli-Matrix（智能运维版本）
- **前期版本**: YYC-Data-Dashboard-Design（数据看盘版本）
- **整合方向**: 以前期版本为功能参考，以核心项目为基准

### 2. 存储架构
- **不启用 PostgreSQL 数据库**
- **支持宿主机存储一体**:
  - PG 15（宿主机）
  - Redis（宿主机）
  - NAS 设备：
    - 向量数据库
    - 时序数据库
    - PG 14
    - MySQL
    - Redis
- **自动识别设备存储**
- **支持写入等操作**
- **同步宿主机权限**

### 3. 功能整合原则
- ✅ 保持当前项目核心稳定
- ✅ 仅整合前端功能模块
- ✅ 跳过后端服务器代码
- ✅ 抽象化存储层，支持多种后端

export class TextToSQLV2 {
  private storage: StorageProvider;

  constructor(storage: StorageProvider) {
    this.storage = storage;
  }

  async query(naturalLanguage: string): Promise<QueryResult> {
    // 1. 解析自然语言
    const sql = await this.parseToSQL(naturalLanguage);

    // 2. 根据存储类型执行
    if (this.storage.type === 'relational') {
      return await this.executeRelational(sql);
    } else if (this.storage.type === 'vector') {
      return await this.executeVector(sql);
    } else if (this.storage.type === 'timeseries') {
      return await this.executeTimeSeries(sql);
    }

    // 3. 返回结果
    return result;
  }
}
