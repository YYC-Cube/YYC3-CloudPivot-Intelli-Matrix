## 一、预算编制原则

### 1.1 预算编制原则

#### 1.1.1 全面性原则

**全生命周期覆盖**
- 覆盖项目启动、规划、执行、监控、收尾全过程
- 覆盖需求分析、设计、开发、测试、部署、运维全阶段
- 考虑项目前期准备、中期实施、后期维护全周期

**全成本要素覆盖**
- 人力资源成本：工资、福利、培训、激励
- 技术资源成本：软件、硬件、云服务、第三方服务
- 基础设施成本：办公场地、网络、设备、水电
- 管理成本：项目管理、质量管理、风险管理
- 应急成本：风险储备、变更储备、应急储备

#### 1.1.2 合理性原则

**基于实际情况**
- 基于项目实际需求编制预算
- 基于市场实际价格编制预算
- 基于团队实际能力编制预算
- 基于技术实际难度编制预算

**考虑风险因素**
- 考虑技术风险，预留技术风险储备
- 考虑进度风险，预留进度风险储备
- 考虑成本风险，预留成本风险储备
- 考虑质量风险，预留质量风险储备

#### 1.1.3 可控性原则

**预算可控**
- 预算分解到可控制的粒度
- 预算分配到可控制的责任人
- 预算监控到可控制的频率
- 预算调整到可控制的范围

**成本可控**
- 成本发生可预测
- 成本变化可监控
- 成本超支可预警
- 成本偏差可纠正

### 1.2 预算编制方法

#### 1.2.1 自上而下法

**方法描述**
- 从项目总体预算开始，逐层分解到各个子项目
- 适用于项目目标明确、经验丰富的项目
- 优点：预算编制快速、便于整体控制
- 缺点：可能忽略细节、不够准确

**实施步骤**
1. 确定项目总体预算目标
2. 分解到各个阶段和模块
3. 分解到各个活动和任务
4. 分配到各个资源和责任人

#### 1.2.2 自下而上法

**方法描述**
- 从各个活动和任务开始，逐层汇总到项目总体预算
- 适用于项目目标不明确、经验不足的项目
- 优点：预算编制准确、考虑细节
- 缺点：预算编制耗时、难以整体控制

**实施步骤**
1. 估算各个活动和任务的成本
2. 汇总到各个阶段和模块
3. 汇总到项目总体预算
4. 与项目总体预算目标对比调整

#### 1.2.3 混合法

**方法描述**
- 结合自上而下法和自下而上法的优点
- 先用自上而下法确定总体预算框架
- 再用自下而上法填充细节预算
- 适用于大多数项目

**实施步骤**
1. 用自上而下法确定总体预算框架
2. 用自下而上法填充细节预算
3. 对比分析，调整平衡
4. 确定最终预算方案
