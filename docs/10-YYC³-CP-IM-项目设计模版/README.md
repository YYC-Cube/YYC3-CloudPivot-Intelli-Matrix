### 使用流程

1. **复制模版**: 从对应的架构层目录中选择合适的模版
2. **修改命名**: 将模版文件名改为实际的文档名称

4. **更新状态**: 将文档状态从 `published` 改为实际状态
5. **移动文档**: 将填充完成的文档移动到相应的阶段目录

## ⚠️ 注意事项

### 文档状态管理

- **模版文档**: 使用 `published` 状态，但内容为模版
- **填充文档**: 根据实际情况使用相应状态（`draft`, `review`, `approved`）
- **已归档**: 使用 `archived` 状态表示已归档的文档

### 版本控制

- 模版版本统一使用 `v2.0.0`
- 填充后更新为实际版本号（如 `v1.0.0`, `v1.1.0`）
- 每次更新递增版本号（语义化版本）

### 命名规范

- 模版命名遵循标准格式：`{序号}-{阶段}-{类型}-{主题}-{文档类型}.md`
- 填充后保持相同命名格式
- 文件名使用中文，便于识别
